## 프로젝트 선정 이유
스프링 부트를 실행하면 "Tomcat initialized with port(s) : 8080"라는 문구가 뜨면서 포트가 열리는데 네트워크 수업에서 TCP/IP 프로토콜 등을 배우며 스프링 부트가 내부적으로 어떻게 포트를 여는지에 대한 이해를 높이기 위해서

### Tomcat
- HTTP와 WebSocket을 처리하기 위한 TCP 기반의 WAS
- 과거에는 Tomcat을 컴퓨터에 설치하고 자바 코드를 Tomcat에 배포해야 했음 → Spring Boot는 Tomcat을 Java 라이브러리(jar) 형태로 서버 내부에 내장하고 있음
- Spring Boot가 실행될 때 "spring-boot-autoconfigure" 모듈이 내장 톰캣 라이브러리가 프로젝트에 존재하는지 확인하고 존재한다면 Tomcat 서버를 자바 객체(Tomcat 클래스)로 직접 생성하는 자동 설정 코드 실행

### ServerSocket
- 자바(JDK)가 태생적으로 가지고 있는 표준 네트워크 통신용 클래스
- 특정 포트 번호를 쥐고, 그 포트로 들어오는 외부 연결 요청을 기다리는 감시원 역할

## 1단계. 8080 포트 열고 브라우저 접속만 확인하기
- **목표**: 서버 소켓을 생성하여 포트를 개설하고, 최초의 TCP 연결을 수립
- **핵심 메커니즘** : 'serverSocket.accept()'는 호출 시 클라이언트의 요청이 올 때까지 스레드가 멈추는 Blocking 상태가 된다. 브라우저가 'localhost:8080'으로 접속하면 OS 커널에서 TCP 3-Way Handshake가 완료되고, 대기 큐에서 연결 정보를 꺼내와 클라이언트 전용 'Socket' 객체를 반환한다.
- **한계점** : 연결(Transport 레이어)은 성공했으나 애플리케이션 레이어의 HTTP 요청 메시지를 읽지 못해, 브라우저는 응답을 받지 못하고 무한 로딩 상태에 빠짐 -> 2단계에서 해결
